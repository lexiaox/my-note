## 1. 定义拆解
CLI 是 **Command Line Interface** 的缩写。

* **Command (命令)**:
    * **核心逻辑**: 祈使句/指令式交互。用户作为指挥官，通过特定的动词（命令）控制系统。
    * **特点**: 精确、无歧义（理想情况下）。
* **Line (行)**:
    * **交互单位**: 以“换行符” (`\n` / Enter) 为信号，作为输入结束和执行开始的标志。
    * **缓冲机制**: 标准输入 (stdin) 通常是**行缓冲 (Line Buffered)** 的。Shell 等待一整行输入完成后才开始解析。
    * **历史渊源**: 源自 **Teletype (TTY)** 电传打字机。早期的输出设备是打印纸，物理上只能一行一行地打印和滚动，无法像屏幕一样随意跳转光标。
* **Interface (界面)**:
    * **位置**: 位于用户与操作系统内核 (Kernel) 之间，通常被称为 Shell (壳)。

## 2. CLI vs GUI 的本质区别
| 特性       | CLI (命令行)           | GUI (图形界面)   |
| :------- | :------------------ | :----------- |
| **操作对象** | 文本流 (Text Stream)   | 图形对象 (WIMP)  |
| **输入方式** | 键盘 (字符序列)           | 鼠标/触控 (坐标点击) |
| **核心优势** | 自动化、组合性 (Pipe)、远程管理 | 直观、学习曲线低、多媒体 |
| **思维模式** | 语言逻辑 (语法、词汇)        | 空间逻辑 (位置、视觉) |

## 3. 为什么是“行”？
因为在 Unix 哲学中，**文本流**是通用的接口。
* 工具 A 产生一行输出 -> 管道 -> 工具 B 接收一行输入。
* `sed`, `awk`, `grep` 等经典工具被称为 "Line-oriented tools" (面向行的工具)，它们的工作方式就是读一行、处理一行、输出一行。
