# 1.  输入
`input()` 和 `sys.argv` 都是用来从用户那里获取输入的方法，但它们的**使用场景、时机和交互方式**有根本的不同。

简单来说：

*   **`input()`**：用于**交互式**输入，程序**运行后**等待用户**手动键入**内容。
*   **`sys.argv`**：用于**命令行参数**输入，在**启动程序时**就将参数传递给程序。

下面我们通过一个表格和详细解释来深入理解它们的区别。

### 核心区别对比表

| 特性       | `input()`           | `sys.argv`                |
| :------- | :------------------ | :------------------------ |
| **输入时机** | 程序**运行中**           | 程序**启动时**                 |
| **交互方式** | **交互式**，程序暂停并等待用户输入 | **非交互式**，一次性传入所有参数        |
| **数据来源** | 标准输入（通常是键盘）         | 命令行（终端/命令提示符）             |
| **数据类型** | 始终返回**一个字符串**       | 返回一个**字符串列表**             |
| **使用场景** | 需要与用户进行**动态对话**的程序  | 需要通过命令行**传递选项、文件名或配置**的程序 |
| **是否阻塞** | **是**，程序会停下来等待输入    | **否**，参数在启动时已就位，程序直接运行    |
| **必需模块** | 内置函数，无需导入           | 需要 `import sys`           |


---

### 详细解释与示例

#### 1. input() - 交互式输入

`input()` 是一个内置函数。当程序执行到 `input()` 时，它会暂停，在控制台显示一个提示符（如果你提供了提示信息），并等待用户键入一些内容并按回车键。然后，用户输入的内容会作为一个**字符串**返回给程序。

**示例：** `input_demo.py`

```python
name = input("请输入你的名字: ")
age = input("请输入你的年龄: ")

print(f"你好，{name}！你今年 {age} 岁了。")
```

**运行方式与输出：**

```bash
$ python input_demo.py
请输入你的名字: 张三
请输入你的年龄: 20
你好，张三！你今年 20 岁了。
```

**特点：**
*   程序与用户有“一问一答”的交互过程。
*   非常灵活，输入内容可以在程序运行过程中动态决定。

#### 2. sys.argv - 命令行参数

`sys.argv` 是 `sys` 模块中的一个列表（list），它捕获了在命令行中运行Python脚本时给出的所有参数。  

**sys.argv打印出来的是list格式的数据**

*   `sys.argv[0]`：永远是**被执行的脚本的名称**。
*   `sys.argv[1]`：第一个命令行参数。
*   `sys.argv[2]`：第二个命令行参数。
*   ... 以此类推。

**示例：** `argv_demo.py`

```python
import sys

print("脚本名:", sys.argv[0])
print("参数列表:", sys.argv)
print("参数个数:", len(sys.argv))

# 检查是否提供了足够的参数
if len(sys.argv) > 1:
    print("第一个参数是:", sys.argv[1])
if len(sys.argv) > 2:
    print("第二个参数是:", sys.argv[2])
```

**运行方式与输出：**

```bash
$ python argv_demo.py hello world
脚本名: argv_demo.py
参数列表: ['argv_demo.py', 'hello', 'world']
参数个数: 3
第一个参数是: hello
第二个参数是: world
```

**特点：**
*   所有参数在程序启动前就已经确定。
*   非常适合自动化脚本、批处理任务，或者需要接收文件路径、配置模式等作为输入的程序。

---

### 如何选择？

*   **当你需要用户根据程序的当前状态做出决定时，使用 `input()`。**
    *   例如：一个游戏，询问玩家“你想进入左边的房间还是右边的房间？（左/右）”。
    *   例如：一个菜单驱动的程序，让用户选择不同的功能。

*   **当你想让用户在执行脚本时就指定好运行方式或数据时，使用 `sys.argv`。**
    *   例如：一个文件处理脚本，需要用户传入输入文件名和输出文件名：`python convert.py input.txt output.pdf`。
    *   例如：一个计算器脚本，直接接收要计算的表达式：`python calc.py "5 + 3 * 2"`。

### 进阶：结合使用与更好的选择

对于复杂的命令行参数（比如可选参数 `-v`， 带值的参数 `--file filename.txt`），直接解析 `sys.argv` 会很麻烦。Python标准库中的 `argparse` 模块是处理命令行参数的专业且强大的工具，强烈推荐在正式项目中使用。

**简单 `argparse` 示例：**

```python
import argparse

parser = argparse.ArgumentParser(description='一个文件处理程序')
parser.add_argument('input_file', help='输入文件的路径')
parser.add_argument('-o', '--output', help='输出文件的路径', default='output.txt')
parser.add_argument('-v', '--verbose', action='store_true', help='显示详细输出')

args = parser.parse_args()

print(f"处理文件: {args.input_file}")
print(f"输出到: {args.output}")
if args.verbose:
    print("详细模式已开启。")
```

**运行方式：**
```bash
$ python my_program.py data.txt -o result.txt -v
处理文件: data.txt
输出到: result.txt
详细模式已开启。
```

### 总结

| | `input()` | `sys.argv` |
| :--- | :--- | :--- |
| **核心思想** | **运行时对话** | **启动时配置** |
| **记住这一点** | 程序问，用户答 | 用户告诉程序怎么做，然后程序自己跑 |

希望这个解释能帮助你清晰地理解两者的区别！

## 1. `%` 格式化 (旧式格式化)

### 基本语法
```python
"格式化字符串" % (值1, 值2, ...)
"格式化字符串" % 值  # 单个值可省略元组
```

### 常用格式符
| 格式符 | 说明 | 示例 | 输出 |
|--------|------|------|------|
| `%s` | 字符串 | `"Name: %s" % "Alice"` | `Name: Alice` |
| `%d` | 十进制整数 | `"Age: %d" % 25` | `Age: 25` |
| `%f` | 浮点数 | `"Price: %f" % 3.14` | `Price: 3.140000` |
| `%x` | 十六进制 | `"Hex: %x" % 255` | `Hex: ff` |
| `%%` | 字面%符号 | `"Discount: 50%%"` | `Discount: 50%` |

### 使用示例
```python
name = "Bob"
age = 30
height = 1.755

# 多个值
print("Name: %s, Age: %d" % (name, age))
# 输出: Name: Bob, Age: 30

# 格式控制
print("Height: %.2f meters" % height)  # 保留2位小数
# 输出: Height: 1.75 meters

print("Number: %05d" % 42)  # 宽度5，不足补0
# 输出: Number: 00042

# 字典形式
data = {"product": "book", "price": 29.9}
print("Product: %(product)s, Price: $%(price).1f" % data)
# 输出: Product: book, Price: $29.9
```

## 2. `.format()` 方法 (新式格式化)

### 基本语法
```python
"字符串{}字符串".format(值)
"字符串{索引}字符串".format(值0, 值1)
"字符串{关键字}字符串".format(关键字=值)
```

### 使用示例
```python
name = "Charlie"
age = 35
salary = 5000.50

# 按顺序填充
print("Hello {}, you are {} years old.".format(name, age))
# 输出: Hello Charlie, you are 35 years old.

# 按索引填充
print("Hello {1}, you are {0} years old.".format(age, name))
# 输出: Hello Charlie, you are 35 years old.

# 按关键字填充
print("Name: {name}, Salary: ${salary:.2f}".format(name=name, salary=salary))
# 输出: Name: Charlie, Salary: $5000.50

# 格式控制
pi = 3.1415926
print("Pi: {:.3f}".format(pi))  # 保留3位小数
# 输出: Pi: 3.142

# 对齐和填充
print("Name: {:>10}".format(name))   # 右对齐，宽度10
print("Name: {:<10}".format(name))   # 左对齐，宽度10  
print("Name: {:^10}".format(name))   # 居中对齐
```

---

### 3. `f-string` (格式化字符串字面值)

### 基本语法
```python
f"字符串{变量}字符串"
f"字符串{表达式}字符串"
```

### 使用示例
```python
name = "David"
age = 28
score = 95.5

# 直接嵌入变量
print(f"Hello {name}, you are {age} years old.")
# 输出: Hello David, you are 28 years old.

# 支持表达式
a, b = 5, 3
print(f"{a} + {b} = {a + b}")
# 输出: 5 + 3 = 8

# 调用方法
print(f"Name in uppercase: {name.upper()}")
# 输出: Name in uppercase: DAVID

# 格式控制
price = 19.9876
print(f"Price: ${price:.2f}")        # 保留2位小数
print(f"Score: {score:.1f}%")        # 保留1位小数
print(f"Hex: {255:#x}")              # 十六进制
print(f"Percent: {0.856:.1%}")       # 百分比格式

# 对齐
print(f"Name: {name:>10}")           # 右对齐
print(f"Name: {name:<10}!")          # 左对齐
```

|------|------------|-------------|------------|
| **可读性** | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **简洁性** | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **性能** | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |
| **表达式支持** | ❌ | ❌ | ✅ |
| **类型安全** | ⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **Python版本** | 2.0+ | 2.6+ | 3.6+ |
| **推荐程度** | 不推荐 | 过渡方案 | **强烈推荐** |

## 常用格式控制符

### 数字格式化
```python
number = 1234.5678

# 通用格式
f"默认: {number}"          # 1234.5678
f"两位小数: {number:.2f}"   # 1234.57
f"千分位: {number:,}"       # 1,234.5678
f"科学计数: {number:.2e}"   # 1.23e+03

# 百分比
f"百分比: {0.75:.1%}"       # 75.0%

# 进制转换
f"二进制: {42:b}"           # 101010
f"十六进制: {255:x}"        # ff
```

### 字符串对齐
```python
text = "Python"

f"左对齐: |{text:<10}|"    # |Python    |
f"右对齐: |{text:>10}|"    # |    Python|
f"居中对齐: |{text:^10}|"  # |  Python  |
f"填充: |{text:*^10}|"     # |**Python**|
```

---

## 最佳实践建议

1. **新项目**：一律使用 **f-string** (需要Python 3.6+)
2. **兼容旧版本**：使用 **.format()** 方法
3. **维护老代码**：了解 `%` 格式化，逐步迁移

### 选择指南
```python
# ✅ 推荐 - f-string (最现代)
name = "Alice"
print(f"Hello {name}!")

# ✅ 可用 - .format() (兼容性好)
print("Hello {}!".format(name))

# ⚠️ 避免 - % 格式化 (已过时)
print("Hello %s!" % name)
```

f-string 以其卓越的可读性、性能和灵活性，成为现代Python开发的首选方案。
