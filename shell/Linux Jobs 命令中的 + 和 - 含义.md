

在 shell 中执行 `jobs` 时，任务 ID 后面的符号表示 **默认操作优先级**：

### 符号解释
- **`+` (Current Job)**: 
  - **含义**: 最近一次被挂起或放入后台的任务。
  - **作用**: 当执行不带参数的 `fg` 或 `bg` 命令时，默认操作此任务。
  - **引用方式**: `%+` 或 `%%`。
- **`-` (Previous Job)**: 
  - **含义**: 倒数第二个被挂起或放入后台的任务。
  - **作用**: 当带有 `+` 的任务结束后，`-` 对应的任务通常会晋升为 `+`。
  - **引用方式**: `%-`。

### 示例
```bash
$ jobs
[1]  + suspended  vim file.txt    # 输入 fg 会打开这个
[2]  - running    python script.py # 备选任务
