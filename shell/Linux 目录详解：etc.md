

`/etc/` 是 Linux 系统的**全局配置文件目录**。

### 核心特征
1.  **含义**: Et cetera (等等) / Editable Text Configuration (可编辑文本配置)。
2.  **内容**: 纯文本的配置文件。没有二进制可执行程序。
3.  **权限**: 普通用户通常只能**读**，只有 Root (sudo) 才能**写**。
4.  **作用域**: **Global (全局)**。修改这里的文件会影响系统上的每一个用户。

### 常见重要文件
| 文件路径 | 作用 |
| :--- | :--- |
| `/etc/passwd` | 用户账户信息列表 |
| `/etc/shadow` | 用户密码文件（加密存储） |
| `/etc/group` | 用户组信息 |
| `/etc/hosts` | 本地主机名解析（IP <-> 域名） |
| `/etc/fstab` | 开机自动挂载硬盘/分区的配置 |
| `/etc/sudoers` | `sudo` 权限配置文件（严禁直接 vim，请用 `visudo`） |
| `/etc/apt/` | 软件包管理器源列表 (Debian系) |
| `/etc/ssh/` | SSH 服务端和客户端配置 |

### 操作建议
- **备份**: 在修改 `/etc` 下的任何文件前，务必备份！
  `sudo cp /etc/config.file /etc/config.file.bak`
- **层级**: 如果你想修改软件配置，先检查是否有**用户级配置**（通常在 `~/.config/` 或 `~/`），优先改用户级的，不要轻易动系统的。
